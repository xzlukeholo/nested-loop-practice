# nested-loop-practice
# Python 巢狀迴圈基礎與實務應用練習

這個儲存庫（Repository）記錄了我學習 Python 巢狀迴圈（Nested Loops）的練習過程，包含基礎圖形輸出、排列組合，以及結合條件控制（`match-case`、`continue`）的實務劃位系統模擬。

## 📂 練習題目說明

### 1. 基礎練習題
* **題目 1：星星矩形** - 使用雙層迴圈印出 4 列 5 欄的星星圖形。
* **題目 2：基本座位表走訪** - 二維清單（2D List）的遍歷練習。
* **題目 3：早餐組合搭配** - 運用巢狀迴圈實現排列組合（食物 + 飲料）。

### 2. 進階挑戰題
* **挑戰題一：靠右對齊的數字直角三角形**
  * **學習重點**：掌握迴圈變數與數學規律的連動（`blank = 5 - row`）。
  * **程式碼演進**：內含初版（調整計數器）與優化版（直接利用 `range(1, 6)` 控制範圍），程式碼結構變得更簡潔，邏輯更直覺！
* **挑戰題二：電影院座位劃位與排他邏輯**
  * **學習重點**：在巢狀迴圈中加入條件判斷。
  * **實作技術**：運用 Python 的 `match-case` 語法進行特定座位的篩選，並利用 `continue` 跳過已售出或不開放的座位。

* **挑戰題三：門市營收分析小助手**
  * **學習重點**：練習使用巢狀迴圈處理「門市 × 日期」這種二維資料結構。
  * **實作技術**：
    * 使用 `range(len(...))` 控制索引位置。
    * 透過 `revenues[store_index][day_index]` 取得指定門市在指定日期的營收。
    * 使用 `store_total` 計算單一門市三天總營收。
    * 使用 `company_total` 計算全公司三天總營收。
    * 使用 `if / elif / else` 根據總營收判斷門市表現。
  * **資料分析概念**：根據我詢問AI的教學跟判斷,感覺這題開始接近實務資料分析中的「分組統計」概念，也就是依照門市分組，計算各門市的總營收與表現狀態。


## 💡 學習心得與反思
在挑戰題一中，我體會到「控制變數」的重要性。初版雖然成功達到效果，但用了較多的輔助變數（`row_count`、`blank_count`）；經過提示後，發現可以直接利用外層迴圈的 `row` 與總長度的數學關係來計算空格，讓程式碼精簡了許多。

在挑戰題二中，練習了如何將現實生活中的業務邏輯（如：設備維修、座位已售）轉換為程式碼中的條件控制，這讓我對邏輯判斷的實務應用更有概念了！

在挑戰題三中，我練習了更接近資料分析情境的巢狀迴圈應用。在這的情境下,題目不只要求我印出資料，而是要我試著去處理操作各個門市的營收數據,這讓我學會如何處理二維清單,同時在二維的資料當中去正確抓取我要的資料,然後進行我們需要的判斷,雖然數據跟規模很小,但我感覺有點碰到數據資料分析的新手大門了。

我學到 `store_total` 和 `company_total` 的位置很重要：

* `store_total` 要放在外層迴圈裡面、內層迴圈前面，因為每一間門市都要重新計算自己的三天總營收。
* `company_total` 要放在所有迴圈外面，因為全公司的總營收需要一路累加所有門市的資料，不能每換一間門市就歸零。

這次練習讓我更理解巢狀迴圈中「變數放在哪一層」會直接影響程式結果，也開始接觸到資料分析常見的分組統計邏輯。
